## 🛠 项目简介

本项目是一个用于 **B 站批量举报视频** 的自动化工具，适用于举报搬运、侵权、低俗等内容，支持多种功能和策略控制。

---

## 🔧 功能特性

* ✅ 自动人机验证
  （参考项目：[Text_select_captcha](https://github.com/MgArcher/Text_select_captcha)）
* ✅ 配合 Clash 客户端自动切换代理，绕过 **412 风控**
* ✅ 支持通过关键词自动搜索目标视频
* ✅ 支持黑名单 / 白名单过滤机制
* ✅ 支持从「稍后再看」列表自动读取举报对象
* ✅ 举报理由采用 **加权随机算法**

---

## 🚀 使用方法

### 一、账号角色说明

* **Reporter（举报账号）**
  实际执行举报操作的账号

* **Collector（采集账号）**
  仅用于采集视频或 UP 信息，不参与任何举报行为

---

### 二、名单规则说明

* **稍后再看列表**

  * 列表内的 **UP 主会自动加入举报名单**

* **黑名单**

  * 黑名单内的账号 **始终会被举报**

* **白名单**

  * 白名单内的账号 **永远不会被举报**
  * 白名单优先级最高

---

### 三、运行步骤

1. 下载 Release，解压到任意目录
2. 双击 `main.bat`
3. 输入 `2`，回车
4. 使用浏览器扫码登录 **Reporter 账号**
5. 再次双击 `main.bat`
6. 输入 `1`，回车
7. 程序开始自动举报

## ⚠️ 注意事项

1. **必须使用支持 Clash 内核的代理工具**

   * 本项目刚需 Clash 内核
   * 用于绕过 B 站风控（如 412 错误）

2. **端口要求**

   * 代理端口：`7890`
   * 控制端口：`9090`

3. **策略组要求**

   * Clash 中必须存在策略组，名称默认为：

     ```
     哔哩哔哩
     ```
   * 程序将通过控制端口自动切换该策略组内的节点

4. **运行前检查**

   * Clash 已启动
   * 控制端口 `127.0.0.1:9090` 可访问
   * 策略组内至少包含一个可用代理节点
